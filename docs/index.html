RetailMind

Data Science Practicum 2 project by Sai Teja Sunku, Regis University.

This is a small project I built to answer one question. Big retailers like Walmart and Costco have data teams that handle forecasting and inventory. Independent shops do not. Can I build one pipeline that any retailer could point at their own sales spreadsheet and get back the same useful output: forecasts, anomaly alerts, drivers, and reorder suggestions.

I tested it on two datasets that look nothing alike. Rossmann store sales (about a million daily rows across 1,115 stores) and Walmart Retail (about eight thousand transactional orders with no store column at all). The same code runs both end to end without changes.

Code

The repository is on GitHub at github.com/ssunku99/Retailmind. The README explains how to run it locally and how the modules fit together. 

git clone https://github.com/ssunku99/Retailmind
cd Retailmind
pip install -r requirements.txt
streamlit run app.py

What the pipeline does

  • Reads the file and profiles it (quality score, missing rates, column candidates).
  • Auto detects which raw column plays each role using name patterns plus dtype.
  • Fills schema gaps. If a dataset has quantity and unit price but no sales column, it derives revenue. If there is no store column, it promotes country or region.
  • Picks daily, weekly, or monthly aggregation based on how dense the data is.
  • Engineers features. Calendar, lags, rolling stats, cyclical encodings. Excludes anything that would leak the target.
  • Trains a LightGBM forecaster with walk forward cross validation and compares against a seasonal naive baseline.
  • Detects anomalies three different ways and explains each one in plain English.
  • Fits a driver model to show which features actually move sales.
  • Turns the forecast into reorder recommendations with a service level and lead time you choose.
  • Answers natural language questions about all of the above.

Verifying it works on other data

I wrote a stress test that generates eleven synthetic datasets imitating common Kaggle retail shapes (Superstore, M5, Olist Brazilian, Indian e-commerce, grocery POS, restaurant chain, SaaS MRR, and so on). All eleven pass every stage end to end. You can re-run it from the repo: 

python tests/stress_test_datasets.py

For a single file diagnostic that pinpoints exactly which stage breaks: 

python diagnose.py path/to/your_file.csv
